# TaskSwitcher

TaskSwitcher — многопользовательская игра на быстроту мышления. Две команды соревнуются в реальном времени: каждый раунд на экране появляется символ (цифра или буква), и игроки должны как можно быстрее выбрать правильный вариант ответа. Побеждает команда, набравшая больше очков по итогу всех раундов.

## Игровые правила

Каждый раунд игрокам показывается **цифра (1–9)** или **буква**. Нужно выбрать один из двух вариантов:

| Символ | Правило | Варианты |
|--------|---------|----------|
| Цифра | Чётное или нечётное? | чётное / нечётное |
| Цифра | Больше 5 или нет? | больше 5 / не больше 5 |
| Буква | Гласная или согласная? | гласная / согласная |

- Правильный ответ приносит **+10 очков** игроку и команде, плюс **бонус за скорость** (до +5 очков)
- Неправильный ответ снимает **-5 очков** с командного счёта
- Игра состоит из нескольких раундов (зависит от формата: 2v2 или 3v3)
- В команду можно добавить **ботов** с настраиваемой точностью ответов

## Возможности

- Регистрация и вход по email/паролю
- Создание комнаты с выбором формата (2v2 или 3v3) и вступление по коду
- Реальное время: Pusher транслирует события раундов, ответов и счёта
- Добавление ботов в команду
- Статистика игрока: матчи, победы, точность, очки
- Рейтинговая таблица: по игрокам и командам

## Структура проекта

```
app/
├── (auth)/           — страницы входа и регистрации
├── (game)/
│   ├── lobby/        — лобби: создание/вход в комнату
│   ├── game/[roomId] — экран матча в реальном времени
│   └── results/[id]  — итоги матча
├── stats/            — личная статистика
├── leaderboard/      — таблица лидеров
└── components/       — Navbar и общие компоненты

server/
├── db/               — схема Drizzle + подключение к PostgreSQL
├── game/
│   ├── engine.ts     — игровая логика (генерация раундов, очки)
│   ├── store.ts      — in-memory состояние комнат, игровой цикл
│   ├── bot.ts        — бот с таймером и настраиваемой точностью
│   └── __tests__/    — unit-тесты (Vitest)
└── trpc/
    └── routers/      — rooms, matches, leaderboard

e2e/                  — E2E-тесты (Playwright)
scripts/seed.ts       — наполнение БД тестовыми данными
```

## Запуск проекта

### Вариант 1 — Docker (рекомендуется)

Запустить всё приложение вместе с БД одной командой:

```bash
npm run docker:up
```

Приложение будет доступно на `http://localhost:3000`.

```bash
# Остановить
npm run docker:down

# Посмотреть логи
npm run docker:logs
```

---

### Вариант 2 — Локальный запуск

#### 1. Клонировать репозиторий

```bash
git clone <url-репозитория>
cd task-switcher
```

#### 2. Установить зависимости

```bash
npm install
```

#### 3. Настроить переменные окружения

```bash
cp .env.example .env
```

Минимально необходимые переменные в `.env`:

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/task_switcher
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your-secret-key-here

PUSHER_APP_ID=...
PUSHER_KEY=...
PUSHER_SECRET=...
PUSHER_CLUSTER=eu
NEXT_PUBLIC_PUSHER_KEY=...
NEXT_PUBLIC_PUSHER_CLUSTER=eu
```

#### 4. Запустить PostgreSQL

```bash
# Только БД через Docker
npm run docker:db

# Или вручную, если Postgres установлен локально
psql -U postgres -c "CREATE DATABASE task_switcher;"
```

#### 5. Применить схему БД

```bash
npm run db:push
```

#### 6. Заполнить БД тестовыми данными (опционально)

```bash
npm run db:seed
```

#### 7. Запустить в режиме разработки

```bash
npm run dev
```

Приложение откроется на `http://localhost:3000`.

## Тесты

### Unit-тесты (Vitest)

Покрывают игровой движок (`engine.ts`), бота (`bot.ts`) и хранилище комнат (`store.ts`):

```bash
npm test
# или в watch-режиме
npm run test:watch
```

### E2E-тесты (Playwright)

Покрывают аутентификацию, лобби, рейтинг и страницу статистики:

```bash
# Установить браузеры (один раз)
npx playwright install chromium

# Запустить все e2e-тесты (приложение должно быть запущено)
npm run test:e2e

# Запустить с UI-режимом Playwright
npm run test:e2e:ui
```
